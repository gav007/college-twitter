# Brand and Copy (Binding Spec)

## Product Name
- Primary name: `Loopfeed`
- Legacy/internal repo name may remain `college-twitter`.

## UI Text
- Browser title suffix: `Loopfeed`
- Nav logo text: `Loopfeed`
- Footer copy (if present): `Loopfeed - lightweight social feed`

## Voice
- Clear, minimal, neutral.
- Avoid meme-heavy or aggressive copy in auth and error flows.

## Route and URL Naming
- Keep existing route paths for compatibility.
- Brand affects text/UI labels, not API path structure.

## Asset Naming
- Keep existing file names unless there is a direct feature requirement.
- Prefer low-risk copy updates over large template refactors.

## Definition of Done
- [ ] Header/nav/title updated to `Loopfeed`.
- [ ] Error/auth copy is brand-consistent.
- [ ] No route breakage from branding changes.
